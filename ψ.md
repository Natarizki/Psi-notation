# Psi (Ψ) Notation for Ordered Complexity Classes

**Version:** 1.0.0 | **Author:** Muhammad Nata Rizki Haynar | **License:** CC BY 4.0

---

## 📐 Formal Definition

A pair of functions \((F_1, F_2)\) belongs to the complexity class \(\psi(g(n))\) if there exist positive constants \(c_1, c_2, c_3\) such that for all sufficiently large \(n\):

\[
c_1 \cdot g(n) \;\le\; F_1(n) \;\le\; c_2 \cdot g(n) \;\le\; F_2(n) \;\le\; c_3 \cdot g(n)
\]

**Interpretation:**  
The term \(c_2 \cdot g(n)\) acts as a **"Bridge"** , mathematically guaranteeing that the upstream layer (\(F_1\)) never asymptotically exceeds the downstream layer (\(F_2\)).

## 🧩 Special Complexity Classes

| Notation | \(g(n)\) | Ideal Application |
| :--- | :--- | :--- |
| \(\psi(1)\) | \(1\) (Constant) | Real-time systems, fixed-latency hardware |
| \(\psi(\log N)\) | \(\log N\) | Binary Search, B-Tree indexing |
| \(\psi(N)\) | \(N\) | Data streaming, ETL pipelines |
| \(\psi(N \log N)\) | \(N \log N\) | Sorting (Merge Sort, Quick Sort) |
| \(\psi(N^2)\) | \(N^2\) | Nested loops, specific DP algorithms |

## 📊 Empirical Validation (Benchmark)

The \(\psi\) notation has been tested against classical notations (\(O, \Omega, \Theta, o, \omega\)) using a Python benchmark. Results show that \(\psi\) successfully detects **ordering violations** that standard notations miss.

**Benchmark Results vs \(g(n) = n\):**

| Function | Classification | Status |
| :--- | :--- | :--- |
| \(n^2\) | \(\omega\) | Far larger (\(f/g \to \infty\)) |
| \(\log_2 n\) | \(o\) | Far smaller (\(f/g \to 0\)) |
| \(5n\) | \(\Theta\) | Equivalent to \(n\) (tight bound) |

**Specific \(\psi\) Tests:**

| Case | Result | Description |
| :--- | :--- | :--- |
| \(F_1 = 2n+5\), \(F_2 = 3n+10\) | ✅ **PASSED** | \(F_1 \le F_2\) for all \(n\). Safe pipeline. |
| \(F_1 = 10n\), \(F_2 = 2n\) | ❌ **FAILED** | \(F_1 > F_2\). Bottleneck detected! |
